# 🏭 نظام إدارة مصنع الصبّ

نظام ويب متكامل لإدارة مصنع صبّ: المواد الخام، الخلطات، الزبائن، المبيعات والفواتير، المصروفات، والتقارير — بواجهة عربية كاملة.

## ✨ المميزات

- **🧱 المواد الخام**: جدول رئيسي (الاسم، الكمية، الوحدة، السعر، المجموع) — منع تكرار المادة، والتوريد يتم عبر جدول حركات فرعي، والكمية الإجمالية تُحدَّث تلقائياً، مع تنبيه عند انخفاض الكمية عن الحد الأدنى.
- **⚗️ الخلطات**: خلطة بمعرّف واسم وتاريخ وكمية منتج نهائي ونسب مكونات محسوبة تلقائياً — وعند التنفيذ تُخصم الكميات من المخزون تلقائياً وتُثبَّت الكلفة.
- **👥 الزبائن**: منع التكرار بالاسم أو الهاتف، مع صفحة تفاصيل تعرض الخلطات المزوّدة وفواتير كل زبون.
- **🧾 المبيعات**: فاتورة تلقائية برقم متسلسل (INV-سنة-رقم)، حساب الكلفة من المواد + هامش ربح، طباعة الفاتورة، وتتبع المدفوع/الآجل.
- **💰 المصروفات**: تسجيل يومي حسب الفئة (رواتب، كهرباء، نقل...).
- **📊 لوحة تحكم وتقارير**: إنتاج، مبيعات، قيمة مخزون، زبائن نشطون، صافي ربح، مخطط مبيعات شهري — مع تصفية حسب التاريخ والزبون.

## 🚀 التشغيل

### محلياً (يعمل فوراً)
افتح `index.html` في المتصفح. البيانات تُحفظ محلياً على الجهاز (localStorage) ويمكن تصديرها/استيرادها من صفحة الإعدادات.

### ☁️ الربط مع Supabase (قاعدة بيانات سحابية)
1. أنشئ حساباً مجانياً في [supabase.com](https://supabase.com) ثم أنشئ مشروعاً جديداً (New Project).
2. من القائمة الجانبية افتح **SQL Editor** ← **New query**، والصق محتوى ملف [`supabase/schema.sql`](supabase/schema.sql) ثم اضغط **Run**، ثم كرر الأمر مع ملف [`supabase/schema2.sql`](supabase/schema2.sql) (الحسابات والصلاحيات والنقل والشركاء والرواتب).
3. من **Project Settings ← API** انسخ:
   - **Project URL** (مثل `https://xxxx.supabase.co`)
   - **anon public key**
4. في النظام: افتح صفحة **⚙️ الإعدادات** والصق القيمتين ثم اضغط **حفظ واختبار الاتصال**.

بعد الربط تُحفظ كل البيانات في السحابة ويمكن فتح النظام من أي جهاز.

### 🌐 النشر على الإنترنت (GitHub Pages)
1. أنشئ مستودعاً جديداً على GitHub وارفع هذه الملفات إليه.
2. من **Settings ← Pages** اختر **Deploy from a branch** ← الفرع `main` ← المجلد `/ (root)`.
3. بعد دقيقة يصبح النظام متاحاً على رابط مثل: `https://username.github.io/repo-name/`

> ⚠️ ملاحظة أمان: السياسات الحالية في `schema.sql` مفتوحة للمفتاح العام لتسهيل البدء. عند النشر العام يُنصح لاحقاً بتفعيل تسجيل الدخول (Supabase Auth) وتشديد سياسات RLS.

## 🔐 الحسابات والصلاحيات

بعد تنفيذ `schema2.sql` يصبح الدخول للنظام بحساب وكلمة مرور، مع ثلاثة أدوار:

| الدور | الصلاحيات |
|---|---|
| 👑 المالك | كل شيء: إدارة الحسابات، الشركاء والأرباح، وكل الأقسام |
| 📋 المدير | متابعة لوحة التحكم والتقارير وكل الأقسام عرضاً فقط |
| 🧮 المحاسب | تعديل المبيعات والزبائن والمصاريف والرواتب والنقل، وعرض المواد والخلطات. لا يرى الشركاء ولا إدارة الحسابات |

**الإعداد الأول (مرة واحدة):**
1. في Supabase: **Authentication ← Sign In / Providers ← Email** ← أوقف خيار **Confirm email** (كي تعمل الحسابات فوراً بدون بريد تفعيل).
2. أنشئ حسابك: **Authentication ← Users ← Add user** ← أدخل بريدك وكلمة مرور وفعّل **Auto Confirm User**.
3. سجّل الدخول في النظام — **أول مستخدم يدخل يُعيَّن مالكاً تلقائياً** 👑.
4. أنشئ حسابات المدير والمحاسب من صفحة **👤 المستخدمون** داخل النظام.

> حماية الشركاء والأرباح وجدول الحسابات مفروضة من جهة الخادم (RLS)، وقيود المدير/المحاسب على بقية الأقسام مطبقة في الواجهة.
